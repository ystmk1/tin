## 기술 스택 (무엇을 위해 무엇을 썼나)

| 기술 | 용도 |
| --- | --- |
| **Vite + TypeScript** | 정적 웹 앱 빌드/개발 서버. 메인 산출물 (`dist/`) |
| **d3-force / -selection / -drag / -zoom** | 노트 연결망 그래프(force-directed) 렌더·드래그·줌 |
| **js-yaml** | 노트 .md의 frontmatter 파싱 |
| **Supabase (@supabase/supabase-js)** | 로그인(Google/Kakao OAuth) + Postgres DB(RLS)로 계정별 노트·서지선택 저장 |
| **Vercel** | 호스팅. 정적 사이트 + `api/` 서버리스 함수 자동 인식 |
| **Vercel Functions (@vercel/node)** | 국립중앙도서관·알라딘 도서 API 프록시 — API 키를 서버에만 두기 위함 |
| **Pretendard (CDN)** | 한글 웹폰트 (라이선스 문제로 커스텀 폰트 대신 채택) |
| **esbuild** | `plugin/`의 옵시디언 플러그인 빌드 (부차적, 현재 메인 아님) |

## 작업 요약

이 프로젝트는 **옵시디언 플러그인 아이디어 → 웹 앱(Vite + Vercel)으로 피벗**하며 발전했고, 최종적으로 **Supabase 기반 계정 동기화**까지 구축됨.

완료된 것:
- **웹 앱 구조 확립**: Vite 정적 사이트가 루트, 옵시디언 플러그인은 `plugin/`로 분리. Vercel 자동 인식.
- **노트 파서**: frontmatter(저자/상태/태그/별점/코멘트) + `##### N`/`##### Np.` 페이지 마커 + `**볼드**` 발췌 + `### 소제목` + 대사 따옴표 처리 + 페이지 갭 정규화.
- **메인 3단 뷰**: 랜덤 볼드 발췌(5글자 이상), d3 그래프(저자/구체 태그로 연결, 검색·필터 dim, hover 라벨), 책등 스택(상태별 색).
- **필터**: 단일 "필터" 버튼 → 팝오버에 상태/별점/저자/태그 다중선택 칩.
- **도서 정보 연동**: 국립중앙도서관 + 알라딘 API를 서버리스로 병합 검색 → 노트에 표지·서지 연결(표지 클릭 시 다시검색/지우기 팝업).
- **Supabase 클라우드**: Google/Kakao 로그인, 서지선택(`note_selections`)·노트(`notes`) 계정별 저장(RLS), 웹 업로드(드래그앤드롭+파일피커)·삭제, 로그아웃 시 양식설명 데모 노트.
- **반응형**: 모바일은 발췌+책등(제목/저자)만, 데스크톱은 풀 레이아웃.

후속 UI/UX 다듬기 (같은 날 이어서):
- **책등 디자인**: 데스크톱 3열(제목/저자·출판사/상태·날짜, fr 고정폭으로 저자 정렬 유지) + 종이 그라데이션 복원. 좌측 상태 띠 = 완독(검정 실선)/중단(연회색 실선)/읽는중(연회색 점선).
- **표지 색상 tint**: 서지정보 연결 시 표지 평균색을 1회 추출→캐싱, 책등 배경에 아주 연하게 깔기 (CORS는 `api/img-proxy.ts`로 우회).
- **노트 관리 = 책등 우클릭 컨텍스트 메뉴**: 삭제하기 / 태그 수정하기(frontmatter `tags:` 블록만 정밀 교체). 패널 안 삭제 버튼은 제거.
- **태그 표시**: `#` 제거 + 하위 카테고리는 `/`로 묶어 경로 칩으로.
- **GitHub 링크**: 헤더 → 최하단 푸터. 업로드 버튼: 헤더 → 그래프 박스 우하단.

브랜치: `Side`(작업) → 매번 `main`에 fast-forward 머지. 둘 다 현재 `5c28d48`.

## 성공/실패 기록

**통한 접근**
- **웹 우선 재구조화**: 플러그인은 Vercel에 못 올림(웹 산출물 없음). Vite 앱을 루트로, 플러그인을 `plugin/`로 옮기니 Vercel이 자동 인식.
- **서버리스 프록시로 키 보호**: 도서 API 키를 클라이언트 번들에 안 넣고 `api/nl-search.ts`에서만 사용. 로컬은 `vite.config.ts` dev 미들웨어가 같은 핸들러 재사용.
- **노트 빌드타임 번들 제거**: `import.meta.glob` 대신 Supabase에서 로드 → 번들 ~300KB→124KB로 급감 + 계정별 동적 노트 실현.
- **동기 캐시 + 비동기 백엔드**: `note-metadata`/`notes`를 메모리 캐시로 앞단 처리해 렌더 경로는 동기 유지, 쓰기는 백그라운드로 Supabase 반영.

**버그와 해결**
- **Vercel "invalid JSON from server" (HTTP 500 FUNCTION_INVOCATION_FAILED)**: 두 가지 원인.
  1. `lib/aladin-search.ts`가 정규식에 **literal 컨트롤 바이트**를 포함해 git/Vercel이 *binary*로 인식 → 함수 빌드 실패. `\xNN` escape로 정상화.
  2. `package.json`의 `"type": "module"` 때문에 Node ESM이 **상대 import에 `.js` 확장자 필수**. `../lib/book-search` → `../lib/book-search.js`로 전부 수정.
- **진단 개선**: 클라이언트가 비-JSON 응답일 때 HTTP 상태·content-type·본문 앞부분을 노출하도록 바꿔 원인 추적이 쉬워짐.
- **따옴표 이중 표시**: CSS `::before/::after`가 곡선 따옴표를 넣는데 원문에도 따옴표가 있어 `""..""`. 렌더 시 양 끝(또는 한 쪽만) 따옴표 제거. (원문 .md는 불변)
- **페이지 파싱 실패**: 실제 노트가 `##### 24p.` 형식인데 파서는 `##### 24`만 인식 → 본문 전부 누락. 정규식에 `p./쪽/page` 접미사 허용.
- **날짜 verbose 출력**: js-yaml이 ISO 날짜를 Date 객체로 파싱 → `String()` 시 "Thu Apr 23 2026 …". `Date` 인스턴스를 `YYYY-MM-DD`로 포맷.
- **책등 저자 정렬 흔들림**: status 텍스트 열이 `auto`라 길이별로 폭이 달라져 저자 위치가 행마다 밀림. 데스크톱 grid를 `fr` 고정비율 3열로.
- **표지 색상 추출 CORS**: 알라딘 이미지를 canvas로 직접 읽으면 cross-origin tainting으로 `getImageData` 실패. **`api/img-proxy.ts`(동일 출처·호스트 허용목록)** 경유로 로드 → 픽셀 읽기 가능. 추출색은 캐싱(localStorage + `note_selections.cover_color`)해 1회만 계산.
- **frontmatter 부분 수정**: 태그 수정 시 js-yaml 전체 round-trip을 쓰면 날짜 등이 재포맷됨 → `tags:` 블록만 정규식으로 정밀 교체해 다른 필드/별점/날짜는 byte 그대로 보존.

**실패/되돌린 것**
- **커스텀 폰트(SM견출명조/신신명조) 배포**: 웹 서빙 라이선스 문제 + 파일 6.5MB. → Pretendard CDN으로 교체.
- **flexbox로 책등 모바일 정렬 시도**: 실제 컨테이너가 `display: grid`라 flex 속성이 무시됨 → grid 자체를 오버라이드해야 했음.
- **Open Library 표지**: 한국 책 커버리지가 거의 없어 "알 수 없음" placeholder만 양산 → 제거하고 알라딘 표지로 대체.

## 주요 결정 사항

- **옵시디언 플러그인 → 웹 앱**: 사용자가 "애초에 웹이었다고 생각하고 재구성" 요청. 플러그인은 `plugin/`에 보존만.
- **Google Drive 대신 Supabase**: Drive는 OAuth 스코프 검수·동기화 복잡도가 큼. Supabase는 Google·Kakao 둘 다 기본 지원 + Postgres + 무료 티어. (단점: 7일 미사용 시 일시정지)
- **노트를 DB 텍스트 컬럼으로** (Storage 버킷 X): md는 텍스트라 컬럼이 단순하고 검색/정렬 쉬움. 200KB 노트도 문제없음.
- **그래프 연결 = 저자 OR 구체 태그**: `소설`·`문학` 같은 광범위 태그는 제외(`GENERIC_TAGS`)하고 `한국문학/중국문학/프랑스문학` 레벨로 연결.
- **상태는 좌측 띠로**: 종이 그라데이션 배경은 복원하고, 읽기 상태는 책등 **좌측 띠**로만 표현(완독=검정 실선/중단=연회색 실선/읽는중=연회색 점선). 표지색 tint는 그 위에 아주 연하게.
- **표지색 추출은 클라이언트 + 프록시**: 서버에서 이미지 디코딩(무거운 deps) 대신, 프록시로 CORS만 풀고 canvas 평균색 계산. 결과만 저장.
- **태그 입력은 평면 태그로 분해**: `/`로 입력해도 한 개의 경로 태그가 아니라 분리된 평면 태그로 저장(그래프가 leaf로 연결하므로 `소설`이 leaf가 되어 제외되는 문제 방지).
- **앱은 Supabase 없이도 동작**: env 비면 localStorage 모드로 폴백 → 설정 전에도 안 깨짐.

## 주의사항 & 교훈

- **PowerShell + git**: git이 stdout 정보를 stderr로 내보내 PowerShell이 빨간 "RemoteException"으로 표시함. **에러 아님.** push/commit 실제 성공 여부는 결과 줄(`a..b branch -> branch`)로 확인.
- **`"type": "module"` + Vercel 함수**: 상대 import는 반드시 `.js` 확장자. TS `moduleResolution: bundler`가 `.js`→`.ts`로 매핑해줘서 dev/build/배포 모두 정상.
- **정규식에 컨트롤 문자 금지**: literal 바이트(`\x00`–`\x1F`)를 넣으면 git이 binary로 분류 → 도구·번들러가 오작동. `\xNN` escape 사용.
- **Supabase SQL 붙여넣기**: 컬럼 정렬용 공백을 넣지 말 것. 특수 공백문자가 섞이면 `syntax error at or near " "`. 일반 공백만.
- **Vercel 배포 전제**: ① env 변수(`NL_API_KEY`, `ALADIN_TTB_KEY`, `VITE_SUPABASE_URL`, `VITE_SUPABASE_ANON_KEY`)를 Production/Preview/Development 모두 설정. ② Production Branch가 실제 push 브랜치와 일치하는지 확인(과거 main이 stale이라 픽스가 배포 안 됨).
- **모바일/데스크톱 분리**: 변경 시 `@media (max-width: 768px)` 양쪽 다 점검. 책등은 grid라 flex 속성 안 먹음.
- **데모 노트 ↔ 실제 노트 구분**: filePath가 `demo/`로 시작하면 데모(삭제 불가).
