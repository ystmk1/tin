## 1) 노트 패널 — 읽기 경험

### 본문 구조 인식

파서가 노트 본문을 이렇게 쪼개서 패널에 표시함:

| 영역 | 트리거 | 패널 표시 |
| --- | --- | --- |
| `comment` | frontmatter `comment` 필드 | 상단 코멘트 박스(전용 스타일) |
| `externalQuote` | 본문 맨 앞 `>` 인용 라인들 | 서평 인용 박스 |
| `preamble` | 서평 ~ 첫 `##### N` 사이 | 도입부 본문(소제목·문단 그대로) |
| `pages` | `##### N` / `##### Np.` / **`##### -p.`** | 각 페이지 섹션 |
| `backlinks` | 다른 노트의 `[[제목]]` / `![[제목]]` | 패널 맨 아래 "이 노트를 인용한 노트" |

`##### -p.`: 페이지 번호가 없는 이북용 마커. 등장 순서대로 1, 2, 3…으로 자동 번호 매김([`parser-core.ts`](src/parser-core.ts)의 `autoPage` 추적). 숫자와 섞여도 마지막 숫자 다음부터 이어붙음.

`%%comment%%`: 옵시디언식 인라인/멀티라인 주석. **파서가 가장 먼저 제거**해서 발췌·임베드·페이지 인덱스 어디에도 안 보임.

### 본문 내 인용박스·임베드

[`renderBodyHTML`](src/view-web.ts) 흐름:

```
HTML escape
  → ### 제목 → .dokki-subheading (블록)
  → 제목 양옆 빈 줄 정리 (마진이 간격을 담당하도록)
  → > 인용 라인 그룹화 → .dokki-blockquote 박스 (빈 줄로 새 박스 시작)
  → ![[Note]] 임베드 → .dokki-embed (재귀, depth 1)
  → ![](url) 이미지 → 삭제 (사용자 요청)
  → [[Note|alias]] 위키링크 → .dokki-wikilink (클릭 시 그 노트로 점프)
  → \n{3,} → .dokki-skip 갭
  → **볼드** → <strong> (nested *italic* 보존)
  → *이탤릭* → <em>
```

위키링크 색은 **차분한 슬레이트 블루**(`--dokki-link: #4a6fa5` / 다크 `#8fb4e8`)로 항상 underline.

### 페이지 룰렛 목차 (좌측 팝오버)

- 5행 룰렛, 중앙=현재 페이지, ±2단계로 페이드/스케일.
- 헤드 바에 작은 카운터 `12 / 240` (tabular-nums, 페이드 전환 동기).
- 드래그 회전·스크롤 동기·**휠 회전(110ms 스로틀)**·페이지 클릭 모두 동작.
- 드래그로 이동 가능 (위치 persist).
- 모바일에선 숨김(스크롤·태그 입력과 충돌).

### 백링크

[`backlinksFor(b, all)`](src/view-web.ts) — 모든 노트의 본문에서 `[[b.title]]` / `![[b.title]]` 참조를 찾아 패널 맨 아래에 클릭 가능한 링크로 나열.

## 2) 태그 시스템

### 표시 = 항목별 별도 칩

이전엔 `tags.join("/")`로 모든 태그를 한 경로로 이어 붙였으나, 이제 **각 frontmatter 태그가 자기 칩**. 단일 태그 안의 `/`(경로)는 유지(`문학/한국문학`은 한 칩으로).

### 자동 KDC 태그 (오버레이)

[`kdc.ts`](src/kdc.ts)에서 청구기호 앞 3자리 → `대분류/중분류` 경로 + `소분류` leaf 두 개.

- `813` → `문학/한국문학` + `소설`
- `840` → `문학/영미문학` (genre 미정 시 path만)
- `1xx` → `철학`, `9xx` → `역사` 등 대분류 매핑 10가지
- 7/8 대분류는 LANG 매핑까지 (한국/중국/일본/영미/독일/프랑스/스페인/이탈리아/기타)

[`effectiveTags(b)`](src/note-metadata.ts) = `frontmatter.tags` ∪ `kdcTagsFromCallNo(meta.callNo)`. **노트 .md 원본은 안 건드림 — 오버레이만**.

### 태그 편집기 (`openTagEditor`)

옛 `window.prompt` 자리에 커스텀 다이얼로그:

- 칩 UI: `×`로 제거, **드래그로 순서 변경**(HTML5 DnD, `is-drop-target` 하이라이트).
- 입력 박스: Enter / `,` → 추가, Backspace 빈 상태 → 마지막 제거.
- **자동완성**: 내 다른 노트들의 기존 태그를 칩으로 추천(상위 24개, 입력어로 필터).
- **추천 분류** 행: 그 노트에 연결된 책의 callNo로 KDC 분류 추천. 저장된 callNo로 즉시 + ISBN으로 라이브 재조회 후 합집합(예전 연결 책에도 추천 나타남).
- 저장 시 별점 태그 보존(`☆☆☆☆`), frontmatter `tags:` 블록만 rewrite (날짜·다른 필드 byte 보존).

### 표시 / 검색 / 그래프 / 필터 일관성

| 위치 | 사용 |
| --- | --- |
| 패널 칩 표시 | `effectiveTags`, `_` → 공백(저장값엔 `_` 유지) |
| 필터 태그 목록 | `effectiveTags` leaf, **위계 정렬**(전체 경로 기준 `localeCompare("ko")`), **전집 태그 맨 뒤** |
| 검색 | `effectiveTags` |
| 그래프 연결 | `effectiveTags` (`GENERIC_TAGS` 제외해 한국문학·해외문학 같은 중분류로 연결) |

## 3) 비문학(조각) 컬렉션

`notes` 테이블에 **`is_fragment boolean default false`** 컬럼 추가(필수 SQL — 아래 마이그레이션 섹션). 분류는 `is_fragment`로만 이뤄지고 `비문학` 태그는 그냥 일반 태그.

웹에서:
- `BookNote.isFragment`로 로드.
- `isNonfiction(b) = b.isFragment === true`. 스택에서 제외하고 nonfictionWrap에 카드 형태로 렌더.
- 카드 = 제목 + 첫 ~140자 미리보기(마크업 stripping). 클릭 시 일반 패널 열림.
- **표지·서지 검색·상태 칩 모두 숨김**(책이 아니라서). 제목 + 별점만.
- 책 스택과 그래프 섹션 사이 28px 간격(전용 단처럼 보이게).
- 드래그 다중선택·우클릭 일괄 삭제 지원(2026-05-29).

플러그인은 vault에서 폴더 두 곳을 보고 분류:
- "책 폴더"(`notesFolder`) 안 → `is_fragment=false`
- "조각 폴더"(`fragmentFolder`) 안 → `is_fragment=true` (우선순위)
- 둘 다 비어 있으면 vault 전체 = 책, 책 폴더만 비어 있으면 조각 빼고 전부 책

## 4) 옵시디언 동기화 플러그인

`plugin/` 새 모드: **변경된 노트만 클라우드에 업로드하는 단일 책무**. 옛 in-vault DoKKi 탐색기 뷰는 제거(`plugin/main.js` 220KB 수준).

### 핵심 동작

1. vault의 .md 전체를 `classifyVault()`로 한 번 훑어 `{book, fragment, skip}` 분류.
2. 클라우드의 `notes.filename`만 가볍게 SELECT(content 안 가져옴 → egress 거의 0).
3. 각 파일 SHA-256 해시 → 로컬 해시 캐시(`syncHashes`)와 비교.
4. **mismatch 또는 cloud에 없음** → 배치 upsert 한 번에 보냄.
5. (옵션) `deleteRemoved` ON이면 cloud에만 있는 row를 한 번의 `.in(filename, [...])`로 삭제.

해시 캐시 키에 fragment 플래그 인코딩: `f:<hash>` ↔ `<hash>`. 파일을 폴더 사이로 이동하면 키가 mismatch → 자동 재업로드되며 cloud의 `is_fragment`도 따라옴.

### 인증

PKCE OAuth + `obsidian://dokki-auth` 커스텀 프로토콜 콜백:
- Supabase Dashboard → Authentication → URL Configuration → Redirect URLs에 `obsidian://dokki-auth` **반드시 추가**.
- 플러그인 설정에서 Google/Kakao 버튼 → 브라우저 → 로그인 → OS가 Obsidian으로 복귀 → `registerObsidianProtocolHandler`가 `code` 받아 `exchangeCodeForSession`.
- 콜백이 막힐 경우 폴백: 웹 DoKKi에 로그인된 상태로 DevTools → Local Storage → `sb-…-auth-token` JSON 복사 → 플러그인 설정의 textarea에 붙여넣고 적용.

설정 탭에 **현재 상태**(`로그인됨: 이메일`)를 표시 — 진단의 첫 단서.

### 한글 폴더명 NFC 정규화

macOS HFS+에서 만든 .md는 자모 분리(NFD)로 저장될 수 있어, 설정 칸의 NFC `조각`과 byte 매칭이 실패함. `inFolder()`에서 양쪽 `.normalize("NFC")` 후 비교.

### 진단

`조각 폴더`가 설정돼 있는데 매칭 0이면 알림에 자동으로 vault 첫 5개 경로 샘플 표시:
```
조각 폴더 "X"와 매칭되는 .md 0건. 실제 vault 경로 샘플: a.md / b/c.md / ...
```
콘솔에도 `[DoKKi sync]` 라벨로 분류 카운트 + 설정값 JSON 로깅.

## 5) 볼드만 모드 + 메모

패널 우상단 `×` 옆에 알약 토글 **`볼드만`**. ON 시:

- 본문(`.dokki-panel-body`)에 `font-size: 0` 캐스케이드 → `<strong>`만 명시적 크기 복구.
- 형광펜 배경 제거(요청).
- 볼드 없는 페이지 섹션 자체 숨김(`:has(.dokki-panel-body strong)` 활용).
- 페이지 사이 64px 간격, 같은 페이지의 볼드끼리는 14px.
- 코멘트 / 서평 / 도입부 / 인용박스 / 임베드 / 소제목 / skip 갭 / 위키링크 전부 hide.

### 메모 팝오버

볼드만 ON 시 **저장된 메모가 있는 볼드는 자동으로 팝오버 떠 있음**(2026-05-29 변경). 빈 볼드 클릭하면 빈 팝오버 열림, 타이핑 → 자동 저장.

- 위치: 데스크톱은 패널 왼쪽, 모바일/좁은 폭은 볼드 위/아래로 플립.
- **스크롤 따라다님** (transition 없이 즉시), 패널 viewport 상하 80px에서 페이드.
- 사이즈 가변: 한 줄로 시작 → `scrollHeight`로 auto-grow, 리사이즈 핸들 없음.
- 내부 패딩 `16px 18px`.
- 같은 볼드에 표시 점(•, 슬레이트블루) — 메모 있을 때만, 볼드만 모드에서만.
- Esc로 닫기, 바깥 클릭은 **빈 메모만** 닫고 저장된 건 유지.

### 클라우드 동기화

[`memos`](src/memos.ts): wishlist 패턴 그대로. 키 = `(file_path, bold_key)`, bold_key = 공백 정규화(`\s+`→스페이스).

- 로컬 캐시(`Memo[]`) 즉시 갱신 → UI 지연 0.
- 클라우드 upsert/delete는 **500ms 디바운스** — 키스트로크마다 round-trip 안 함.
- 로그인 시 localStorage 메모를 cloud로 1회 migrate(중복 무시 모드).

## 마이그레이션 (DB)

| 시점 | SQL | 동기 |
| --- | --- | --- |
| 위시리스트 cloud | `create table public.wishlist (user_id, wish_id, title, author, created_at, primary key (user_id, wish_id)) + RLS` | 4가지 RLS 정책 |
| KDC 청구기호 | `alter table public.note_selections add column if not exists call_no text;` | 책 연결시 callNo 보관 |
| 조각 분류 | `alter table public.notes add column if not exists is_fragment boolean not null default false;` | 컬렉션 분기 |
| 메모 | `create table public.memos (user_id, file_path, bold_key, text, updated_at, primary key (user_id, file_path, bold_key)) + RLS` | 4가지 RLS 정책 |

전부 `if not exists` 안전 패턴. 컬럼이 없으면 해당 기능만 localStorage 폴백으로 떨어지고 앱은 안 깨짐(metadata cloud upsert는 row 단위 실패).

## 자잘한 UI/파서 다듬기 (시간 순)

- 상단 그래프뷰 **Three.js 우주/은하** 디자인 — 별 크기·밝기는 별점, 거리는 관계성(저자/태그/`[[참조]]`). filter UI에 연결성 모드 토글(off/author/tag/both).
- 그래프 필터 팝오버 **드래그 이동**.
- 책등 우측 띠 색을 OKLCH로 — 표지 hue 유지하되 luminance만 상태별로 분리(완독/중단/읽는 중 구분).
- **마키 드래그 다중선택** + 우클릭 일괄 삭제(책 스택 + 조각).
- 모바일에선 마키 OFF(스크롤과 충돌).
- 노트 패널 열림 시 **배경 스크롤 잠금**(html에 `.dokki-scroll-lock`).
- 위시리스트 제목 자동 정리: 같은 제목 노트가 생기면 자동 제거.
- 위시리스트 우클릭 삭제 + 기존 `×`도 유지.
- 사이드바 → **노트 최하단 + GitHub 푸터 위**로 위시리스트 이동.
- 검색 / 필터 / 그래프 / 칩 표시 등 **태그 관련 경로 일관성** 정리.
- 서평 인용 박스 아래 여백 24 → 60 → 72px (도입부와 충분히 분리).
- 소제목 위 여백 1.9em → 3.8em (위는 2엔터, 아래는 1엔터 느낌).
- 상단 본문 인용 양식 끝나고 본문 사이 간격 명시화.
- `> 인용`이 본문에서 박스로 보이게 (빈 줄로 새 박스 시작).
- `![[note]]` 임베드 내부의 `**볼드**` 렌더 → renderBodyHTML 재귀.
- `![]()` 마크다운 이미지 / `![[그림.png]]` 이미지 임베드 = 표시 안 함(사용자 요청).
- 발췌 최소 길이 6자 / 2단어 미만 skip, 다중 줄 볼드 발췌 newline 보존.
- 한쪽 `**`만 있는 단락 자동 balancing (`fixStrayBold`).
- `**a *b* c**` 같이 볼드+이탤릭 중첩 정확히 렌더.
- 노트 다음 패널에서 **발췌 제목 클릭 시 그 줄 화면 가운데**로 스크롤.
- 다중삭제 메뉴(`openMultiDeleteMenu`).
