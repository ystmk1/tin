# DoKKi

독서 발췌 노트를 위한 옵시디언 플러그인. 메인 화면은 세 층으로 구성됩니다.

1. **랜덤 한 줄** — 노트 본문의 `**볼드**` 텍스트 중 무작위 하나 + 제목 / 저자 / 페이지
2. **나무 그래프** — 노트 간 연결을 force-directed로 표시. 연결도 높은 노드가 굵고 중심에서 가지처럼 뻗어나감. 드래그/줌 가능.
3. **책등 스택** — 한 권당 한 줄. 최근 읽은 책이 위, 과거가 아래.

## 노트 양식 (파서가 기대하는 형식)

```markdown
---
author: 저자명
status: 완독            # 또는 "읽는 중" / "142페이지 중단"
start_date: 2025-11-02
end_date: 2025-11-09
tags:
  - 문학/한국문학/소설
  - SF
publisher: 출판사
comment: 짧은 코멘트
---

> 책 외부의 글 (서평, 작가의 말 등). 첫 페이지 마커 전까지만 인식.

##### 12
페이지 본문...
**메인 화면 랜덤 발췌 후보가 되는 볼드 문장.**

##### 47
다음 페이지 본문. 빈 줄, 페이지 내 공백은 그대로 보존됩니다.
```

### 파싱 규칙

- 페이지 마커: `##### <숫자>` (그 줄 단독).
- 다음 페이지 마커 전까지 모두 해당 페이지 본문. 연속 페이지를 합쳐 쓴 경우 첫 페이지 번호로 묶입니다.
- `**…**` 안의 짧은 텍스트가 메인 화면 한 줄 발췌 후보로 인덱싱됩니다.
- 상단 `> …` 인용 블록은 책 외부 자료(서평/작가의 말)로 분류되어 발췌 대상에서 제외됩니다.
- 페이지 내 공백 줄(엔터)은 원본 그대로 유지되어 나중에 본문 보기를 만들 때 활용됩니다.

## 연결 기준 (필터)

- **저자**: 같은 `author` 값을 가진 노트끼리 연결.
- **태그**: 태그 경로의 최하위 카테고리가 같은 노트끼리. 예) `문학/한국문학/소설`과 `문학/한국문학/시`는 `한국문학`이 아니라 각각 `소설`/`시`가 잎이므로 연결되지 않음. 가장 깊은 경로의 잎을 책의 대표 잎으로 사용.

상단 토글로 두 기준을 즉시 전환할 수 있습니다.

## 개발

```powershell
npm install
npm run dev        # watch 빌드. DOKKI_VAULT 환경변수가 있으면 자동 배포.
npm run build      # production 빌드 (main.js 갱신)
```

### 테스트 vault에서 켜기

1. `test-vault/`를 옵시디언에서 vault로 열기.
2. 설정 → 커뮤니티 플러그인 → 안전 모드 끄기 → 설치된 플러그인에 **DoKKi**가 보입니다.
3. 활성화 → 좌측 리본의 책 아이콘 또는 명령 팔레트 `DoKKi 탐색기 열기`.

### 본인 vault에 배포

방법 A — 수동: `main.js`, `manifest.json`, `styles.css` 세 파일을 `<your-vault>/.obsidian/plugins/dokki/`로 복사.

방법 B — 자동: 개발 중 자동 배포가 편하면

```powershell
$env:DOKKI_VAULT = "C:\path\to\your\vault"
npm run dev
```

esbuild가 빌드할 때마다 위 세 파일을 vault의 `.obsidian/plugins/dokki/`에 복사합니다.

설정 탭에서 **노트 폴더** 경로를 지정하면 그 폴더 아래만 파싱합니다. 비우면 vault 전체.
